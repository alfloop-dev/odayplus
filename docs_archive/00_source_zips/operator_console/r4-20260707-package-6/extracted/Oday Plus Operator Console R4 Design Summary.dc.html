Network · Expansion Flow Stepper（R4 統一資料）
① 找區域 Find Areas→ ② 物件雷達 Listing Radar→ ③ 候選點 Candidate→ ④ SiteScore→ ⑤ 比較 Compare→ ⑥ 審核 Review
Golden flow（R4 資料）：信義松仁生活圈（86）→ L-2024 591｜信義區松仁路 18 坪店面（NT$58,000）→ CS-1001 信義松仁候選點 → SiteScore 82 GO（回本 22 個月）→ 加入比較 → 送審 → 審核決策。每步可點擊切換 tab，顯示 current／completed／next／blocked（缺資料）；同一物件的租金與 ID 在所有頁面一致。

② 物件雷達 Listing Radar
9 個物件來源（591／樂屋／仲介 feed／CSV／人工…）。L-2024：NT$58,000／月 · 18 坪 · 1F 臨路 · Geocode 0.94 · 硬規則 3/3 · 唯一物件。CTA＝轉為候選點（建立 CS-1001，狀態 Ready for SiteScore，自動切至候選點工作台）；重複物件可合併（L-2029 同址）、硬規未過封存（L-2030 超坪）。選區 chip 保留於頁面上方，可清除。
③ 候選點資料完整度 Gate
地址定位 ✓ 租金 ✓ 坪數 ✓ 樓層 ✓ 用途 ✓ Geocode ≥ 0.80 ✓ Hard rule ✓ 來源可信度 ✓ 現勘 ⚠ 仲介聯絡 ⚠ 照片附件 ⚠
缺地址／Geocode → 要求人工確認地址（鎖評分）；缺租金坪數樓層 → 補資料；硬規則 fail → 標記不適合；資料齊未現勘 → 可評分＋降信心警示。狀態機：Draft → Need Data → Ready for SiteScore → Scoring → Evaluated → Shortlisted → Pending Review → Approved／Rejected／On Hold／Expired。

⑤ 比較 · 系統推薦　⑥ 審核 · 決策規則
推薦：信義松仁（GO 82）；備選：板橋府中（WAIT 76）；不建議：大安和平（REJECT 49）。CTA：送審信義松仁 · 保留備選 · 封存 · 比較報告 preview。
Review 決策 dialog：原因必填寫入 Decision Log；Approve GO → Approved；Approve WAIT → 必填條件，Candidate = On Hold；Return → 必填需補資料，Candidate = Need Data；Reject → Rejected；Override → 覆寫理由＋風險確認 checkbox。決策後 Candidate／Review 狀態同步。
Growth · 三個建立入口＋Draft Builder
＋建立離峰促銷（平日 10:00–14:00 · 洗烘 9 折）／＋建立會員召回（60 天未回訪 · LINE 券）／＋建立 PriceOps 測試（現價→建議價 · 含回滾條件）。
五步 Builder：基本設定→客群／時段→預估效益→風險／衝突（重疊 · PriceOps 衝突 · 預算 · 打擾 · 主管核准）→送核准。送核准後 Pending Approval 進 Govern 核准中心；核准 → Scheduled → Running → Observing → Outcome Ready（Effective／Ineffective／Inconclusive）。Lifecycle 決策寫入 Decision Log。
Govern · 治理稽核
核准中心：Growth 活動核准 · Network SiteScore 審核 · Store Ops 退款批次 · Evidence 匯出，退回／駁回必填理由。Decision Log：系統建議 vs 最終決策＋理由＋決策者＋模型版本＋資料快照。Audit Trail：Issue 操作、Camera 調閱（含目的）、核准、Network／Growth 行為，隱私敏感操作標記。
